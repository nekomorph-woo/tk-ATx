# 字数统计与阅读时间

## 功能描述

在状态栏或编辑器底部显示字数、字符数、段落数和预计阅读时间。

## 为什么做？

这是低成本、高感知的写作辅助功能。对文章、文档、笔记用户都实用。

## 收益

- 帮助用户控制文档长度。
- 增强写作工具感。
- 实现风险低，适合作为早期小功能。

## 基础设施支持情况

- 现有依赖：Milkdown listener、ProseMirror doc text extraction。
- 可能新增代码：统计器、status bar tile、配置项。
- 可能使用 Pulsar API：status-bar service，可通过 consumedServices 集成。
- 暂不需要新增第三方依赖。

## 实现要点

- 文档变化时 debounce 统计。
- 中文按字符或词策略需要可配置。
- 阅读时间默认按中文/英文混合规则估算。
- 如果 status-bar 服务不存在，优雅降级为不显示。

## 风险与注意事项

- 大文档统计需要防抖。
- 中文“字数”和英文“word count”规则不同，需要明确展示名称。
