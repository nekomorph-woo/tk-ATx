# 脚注编辑

## 功能描述

增强 GFM footnote 的可视化编辑，支持脚注引用跳转、定义编辑、引用和定义之间互相定位。

## 为什么做？

脚注在长文、学术笔记、技术说明中常见，但源码编辑体验割裂。WYSIWYG 可以让引用和定义之间的关系更清晰。

## 收益

- 提升长文档注释体验。
- 减少脚注引用和定义错配。
- 完善 GFM 支持范围。

## 基础设施支持情况

- 现有依赖：Milkdown GFM preset。
- 可能新增代码：footnote scanner、引用浮层、定义跳转 command、重复 id 校验。
- 暂不需要新增第三方依赖。

## 实现要点

- 点击脚注引用显示定义内容摘要和跳转按钮。
- 在定义处提供返回引用按钮。
- 支持创建新脚注并自动生成 id。
- 保存时保持 GFM footnote 语法。

## 风险与注意事项

- GFM footnote schema 在 Milkdown 中的可扩展点需要先验证。
- 多引用同一脚注时返回位置策略需要明确。
